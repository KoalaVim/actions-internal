# AI Conflict Resolution for Plugin Fork Rebasing

## Overview

Add automated AI-powered conflict resolution to the `rebase-plugin.yml` workflow. When a rebase against upstream fails, the AI re-implements the fork's custom commits on top of the current upstream, then opens a PR for human review.

## Constraints

- Must work within GitHub Free plan (no paid features)
- No additional API keys or secrets
- Uses GitHub Models API (GPT-4o-mini) authenticated with the existing app-generated token

## Workflow Flow

```
Rebase attempt
  |
  +-- Success --> Update lockfile PR (existing, unchanged)
  |
  +-- Failure --> Abort rebase
                  |
                  +-- Identify fork's custom commits (merge-base..HEAD)
                  +-- Capture fork's combined diff
                  +-- Reset branch to upstream/<branch>
                  +-- Collect current upstream versions of affected files
                  +-- Call GitHub Models API to re-implement fork changes
                  +-- Apply AI output, commit on new branch
                  +-- Open PR in fork repo for human review
                  |
                  +-- If AI fails at any point --> Fall back to issue + dashboard comment (existing behavior)
```

## Technical Details

### Identifying Fork Commits

After a failed rebase and `git rebase --abort`:

```bash
MERGE_BASE=$(git merge-base HEAD upstream/$UPSTREAM_BR)
FORK_DIFF=$(git diff $MERGE_BASE..HEAD)
FORK_LOG=$(git log --oneline $MERGE_BASE..HEAD)
```

### Collecting Upstream File State

For each file touched by the fork's commits, read the current upstream version:

```bash
AFFECTED_FILES=$(git diff --name-only $MERGE_BASE..HEAD)
for file in $AFFECTED_FILES; do
  git show upstream/$UPSTREAM_BR:$file
done
```

### GitHub Models API Call

Endpoint: `https://models.github.ai/inference/chat/completions`

Authentication: `Authorization: Bearer $GH_TOKEN` where `GH_TOKEN` is `steps.generate_token.outputs.token` (the Koala app token, not the default `GITHUB_TOKEN`)

Model: `openai/gpt-4o-mini` (128k context, sufficient for Neovim plugin Lua files)

One API call with:
- The fork's combined diff (what the fork changed)
- The current upstream versions of affected files
- Prompt instructing the AI to re-implement the fork's changes on the current upstream code
- Instruction to output only the modified file contents in a parseable format

### Prompt Structure

```
You are re-implementing changes from a forked repository onto the current upstream code.

Here are the changes the fork made (as a unified diff):
<fork_diff>

Here are the current upstream versions of the affected files:
<file: path/to/file.lua>
<content>
</file>

Re-implement the fork's changes on top of the current upstream code.
Output ONLY the modified files in this exact format:

--- path/to/file.lua ---
<full file content>
--- END ---
```

### Branch and PR Creation

```bash
BRANCH_NAME="ai-rebase-$PLUGIN_NAME-${NEW_SHA:0:7}"
git checkout -b $BRANCH_NAME
git add -A
git commit -m "chore: AI re-implementation of fork changes on upstream"
git push origin $BRANCH_NAME

gh pr create \
  --repo $GITHUB_REPOSITORY \
  --title "AI Rebase: $PLUGIN_NAME" \
  --body "Automated AI re-implementation of fork changes.

Original fork commits:
$FORK_LOG

Files modified by AI:
$AFFECTED_FILES

This PR was generated by AI and requires human review."
```

### Failure Handling

If any of the following occur, fall back to the existing issue + dashboard comment behavior:
- GitHub Models API returns a non-200 status
- API response is empty or unparseable
- Any error during file writing or git operations

No validation of code correctness is performed. The human reviewer on the PR is responsible for verifying the AI's output.

## What Changes in the Workflow

- The existing "Handle Failure" step is preserved as a fallback
- New steps are inserted between "Attempt Rebase" (failure) and "Handle Failure":
  1. "Extract Fork Changes" - captures diff and affected files
  2. "AI Re-implementation" - calls GitHub Models API
  3. "Apply and Open PR" - writes files, commits, pushes, opens PR
- The "Handle Failure" step's condition changes to: rebase failed AND AI resolution failed
- The success path and lockfile update remain unchanged

## No New Secrets

The GitHub Models API accepts the same token generated by `actions/create-github-app-token`. No new repository secrets are needed.
